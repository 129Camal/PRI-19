Abstract:

MyTwitter

Basicamente para resolver este exercicio comecei por fazer uns templates em pug para visualizar.

Após isso, desenvolvi umas funções no routeador(GET e POST) para receber os pedidos do cliente. Para além disso também utilizei JQuery e AJAX para atualizar a página do cliente sem necessidade de pedidos. Quando o post é aplicado, o servidor recebe a informação mas não precisa de atualizar o cliente.

Foi criada uma base de dados que é um ficheiro JSON onde o servidor guarda a informação quando recebe um POST do cliente e o cliente recebe a informação quando faz um pedido.

Tentei implementar o botão do gosto mas não correu bem, contudo ele é inserido mas não atualiza o número de gostos.

MyStrava

Para resolver este exercício comecei por aplicar a função que o professor forneceu para obter o dataset.

Para otimizar a inserção na base de dados (MongoDB), arranjei o dataset, eliminando a lista JSON e todos os dados não necessários, deixando apenas a distância, o tipo, a data e o tempo de movimento.

Após isso foram criados os modelos e os controladores. Após isso, foi criada a rota API para poder receber os pedidos do cliente e responder, com informação válida como pedido.

Para correr esta aplicação terá que se ligar à base de dados MongoDB, utilizando uma database com nome "myStravazinho" e com a colecção "informacao".
